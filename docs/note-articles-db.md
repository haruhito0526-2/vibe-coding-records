### 2026-05-06 追記：パイプライン設計と段階的検証計画

#### ハイブリッド設計の確定
シーンごとに素材ソースを切り替える方針。素材ソース4種：

| 種別 | ツール | コスト感（5秒） | 用途 |
|---|---|---|---|
| A. Seedance系 | Higgsfield: GPT image2 + Seedance 2.0 | 12+68=80クレ（+68で15秒、+68で30秒） | オープニング・CTA等の印象シーン、破綻少ない |

| C. 記事画像 / キャプチャ | playwrightでURL→画像、または記事画像流用 | ゼロ | 機能の実機説明、最も誠実 |
| D. Remotion単体 | テキスト＋カラーアニメーション | ゼロ | つなぎシーン |

#### プロンプト強制付与（no text徹底）
画像・動画生成のプロンプトには下記を自動付与してRemotion字幕レイヤーと干渉させない：
`no text, no captions, no subtitles, no signs, no logos, no letters, no UI elements, no watermarks`

#### 検証ロードマップ
- **第1回（次セッション）**：記事内画像の利用＋操作シーンの自動キャプチャ。**Higgsfield非使用の最小構成**で完成させる
- **第2回**：訴求の一部を **Klingパターン**（Nano Banana Pro + Kling 3.0）の動画に置き換え
- **第3回**：重要シーン（オープニング・CTA）を **Seedanceパターン**（GPT image2 + Seedance 2.0）で作成

#### パイプライン構成（4ステップ）
```
[1] scene-config.ts 編集（手 or AI）
[2] npm run narration:split   # 既存。ナレ生成→無音検出→分割
[3] npm run assets:generate   # 新規。source種別で分岐（higgsfield CLI / playwright / ローカルファイル）
[4] npm run render            # Remotion合成
```

#### コスト管理
`npm run assets:cost` で事前に各シーンの `higgsfield generate cost` を集計→OK後に `assets:generate`。暴発防止。

### 2026-05-07 追記：v6→v14のブラッシュアップと第1.5回（動画キャプチャ自動化）

#### v7：シーン03に動画キャプチャを導入（=第1.5回）

v6まで全シーンが静止画＋Remotion単体だったが、シーン03（サムネイルキャプチャ）だけ実機操作の動画キャプチャに置き換えた。Playwright（Python）でブラウザを自動操作 → 動画ファイルアップロード → 再生 → スペースキー押下 → サムネがプールに追加される、という14秒の操作シーケンスを録画 → ffmpegでmp4変換。Remotion側で `startFromSec` / `endAtSec` / `playbackRate` を使って中盤の11秒だけ1.4倍速で表示し、ナレーション尺に合わせ込み。

工夫したのは「擬似マウスカーソル」をJSでDOMに注入するアプローチ。OS描画のマウスはPlaywrightの録画には映らないため、白いSVG矢印＋クリック時のリップル効果を動的に注入することで「操作している感」を出した。最初IIFE化を忘れて1回目はカーソルが映らない詰まりを経験。

#### 学んだこと

- シーン跨ぎで「持続させたい要素」は専用 Sequence か、最上位 AbsoluteFill 直下に置く必要がある。そのとき他のシーンの AbsoluteFill より「後」にレンダリングしないとシーン背景に埋もれる
- ロゴや配色の調整は実機レンダリングして見るしかなく、コードの数値だけでは判断できない（ハルさんの目で見ながらの微調整が前提）
- バージョン管理は通し番号で別ファイル保存。上書きすると過去版が取れなくなる（v6を1回上書きして再生成した反省）

ハルさんの希望（既存ナレ音声を保持・サブスク追加なし・後払い／使わなければゼロ円）を満たすツールを探して、6つを比較検討：

### 2026-05-07 追記：Phase2 完了（v17〜v20）

#### 方針の大転換：音声と映像の分離

v16 は「リップシンク動画ファイルそのものを 1本で再生」していたため、ワイプの映像と音声が一蓮托生だった。v17 で発想転換：

- **音声**：リップシンク動画ファイルを `<Audio>` タグで参照（音声トラックだけ抽出再生）
- **映像（ワイプ用）**：同じファイルを `<OffthreadVideo muted>` で参照、`startFrom` を別タイミングに

これで「シーン01〜06 期間にリップシンク音声を流しつつ、シーン02〜06 期間だけ顔ワイプを表示」という分離が実現。Remotion の Audio コンポーネントは MP4 ファイルもそのまま音声ソースとして扱える。

#### リップシンク動画の音声構造解析

ffmpeg の silencedetect で 0.8 秒以上の無音区間を検出し、各シーンセリフの開始秒を特定：

| シーン | リップ動画内の開始秒 | mp3 尺 | リップ尺 |
|---|---|---|---|
| 01 | 0.97s | 4.26s | 3.97s |
| 02 | 4.94s | 7.78s | 8.47s |
| 03 | 13.41s | 6.84s | 7.30s |
| 04 | 20.71s | 6.45s | 7.38s |
| 05 | 28.09s | 3.60s | 4.57s |
| 06 | 32.66s | 3.87s | 4.43s |
| 07 | 37.09s | 4.60s | 5.58s |

シーン尺を「mp3 の duration」基準から「リップシンク内の各セリフ間隔」基準に再計算。SCENE_TAIL_FRAMES（各シーン末尾の余韻 1秒）はリップシンク連続再生のため 01〜06 では適用しない。

#### v17→v20 の微調整履歴

- **v17**：レイアウト初版（右上テキスト＋顔ワイプ 480、顔倍率 270%）。リップシンク音声は シーン01 の 0 フレーム目から流す
- **v18**：シーン00 ロゴ拡大（100→160）＋字幕削除、シーン01 を 1 秒延長して音声・字幕に 1 秒遅延（唐突感緩和）、シーン02 以降のタイトル枠を `right:80, width:480, textAlign:center` に変更して顔ワイプ center と完全一致、AnimatedTitle に `\n` 改行対応、シーン02「ブラウザだけで\n一括生成」/シーン05「VideoObject\nスキーマ生成」
- **v19**：シーン00 ロゴ 160→130
- **v20**：シーン00 ロゴ 130→120（「ほんと微妙にキャラにかぶる」の 10px 詰め）

#### Phase2 の学び・記事ネタ候補

- **「リップシンクの音声と映像を分離する」発想転換**：1ファイルで持つと自由度が低い。Remotion の Audio + OffthreadVideo の組み合わせで、同じファイルを別タイミングで参照できる
- **無音検出による尺の自動算出**：mp3 ベースで作った尺感をリップシンクに合わせ直すとき、ffmpeg silencedetect が決定的に効く（耳で測らずに済む）
- **シーン01 の 1 秒遅延というディテール**：シーン00（Seedance 動画）のあと急にナレが続くと唐突感がある。ロゴ・タイトルは 0 秒目から表示しつつ、音声と字幕だけ 1 秒遅らせると視聴体験が大きく改善
- **ロゴ高さの 10px 単位の調整**：120/130/160 で印象がはっきり変わる。「キャラにかぶる/かぶらない」のギリギリは数値で詰めるしかない
- **テキストと顔ワイプの center 揃え**：単純なレイアウト原則だが、ワイプを center 1600 として枠の width と right で center を一致させる（textAlign:center で文字を寄せる）と整って見える
- 最終成果物：`AI内製検証/remotion-test-output/video-aio-seo-tool-v20.mp4`（46.8 秒、24MB）
